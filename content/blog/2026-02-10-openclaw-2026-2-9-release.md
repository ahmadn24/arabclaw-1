# إطلاق أوبن كلاو (OpenClaw) 2026.2.9: تحديث شامل مع تطبيق iOS وتحسينات الاستقرار

**ما ستتعلمه في هذا المقال:** يستعرض هذا المقال أبرز الميزات الجديدة في إصدار أوبن كلاو (OpenClaw) 2026.2.9، بما في ذلك تطبيق iOS التجريبي الجديد ودعم Grok للبحث الذكي و8 إصلاحات رئيسية لـ Telegram. ستتعرف على كيفية التحديث والاستفادة من التحسينات الجديدة.

أطلق فريق أوبن كلاو (OpenClaw) اليوم الإصدار 2026.2.9، وهو تحديث رئيسي يجلب ميزات جديدة مثيرة وإصلاحات شاملة للاستقرار. هذا التحديث يمثل قفزة نوعية في تجربة المستخدم والموثوقية.

## 🎉 أبرز الميزات الجديدة

### 1. تطبيق iOS التجريبي: أوبن كلاو (OpenClaw) في جيبك

أخيرًا! وصل أوبن كلاو (OpenClaw) إلى أجهزة iPhone وiPad مع إطلاق التطبيق التجريبي Alpha. الآن يمكنك:

- **التحكم في الهاتف عن بُعد**: إدارة جهاز iOS من أوبن كلاو (OpenClaw) على الكمبيوتر
- **المزامنة السلسة**: نظام Onboarding متكامل مع Setup Code
- **Node Pairing**: ربط الأجهزة بسهولة عبر Telegram مع أمر `/pair`

هذه خطوة كبيرة نحو نظام أوبن كلاو (OpenClaw) الموحد عبر جميع الأجهزة. التطبيق متاح للاختبار المبكر للمساهمين.

```bash
# قريباً: تثبيت من TestFlight
openclaw nodes pair --device ios
```

### 2. محرك البحث الذكي Grok: قوة xAI في خدمتك

تم دمج Grok من xAI كمزود بحث في أداة `web_search`. الآن يمكنك:

- البحث مع الاستشهادات المضمنة (Inline Citations)
- نتائج بحث أكثر دقة وسياقية
- تكامل سلس مع نماذج xAI

```javascript
// مثال استخدام
await web_search({ 
  query: "أفضل ممارسات DevOps 2026",
  provider: "grok"
});
```

### 3. إدارة Agents من واجهة الويب

أصبح بإمكانك الآن إدارة Agents مباشرة من Web UI:

- **إنشاء Agents جديدة**: `agents.create`
- **تعديل الإعدادات**: `agents.update`
- **حذف Agents**: `agents.delete`

واجهة بديهية لإدارة جيش من AI Agents بدون سطر أوامر!

### 4. تحسينات BlueBubbles: iMessage أفضل من أي وقت

تنظيف شامل لدعم BlueBubbles (iMessage على الأندرويد/ويندوز):

- كود أنظف وأكثر استقرارًا
- أداء محسّن للرسائل
- دعم أفضل للمحادثات الجماعية

## 🔧 إصلاحات Telegram الشاملة

الإصدار 2026.2.9 يحمل **8 إصلاحات رئيسية** لـ Telegram، مما يجعله القناة الأكثر استقرارًا:

### 1. نظام الاقتباسات المحسّن
- **تحليل أقوى**: معالجة معززة للاقتباسات المتداخلة
- **تجنب الأخطاء**: لا مزيد من `QUOTE_TEXT_INVALID`
- **حفظ السياق**: الردود تحافظ على السياق الكامل

### 2. إرسال استباقي أكثر ذكاءً
استرداد تلقائي عند استخدام thread IDs قديمة:
- محاولة أولى مع `message_thread_id`
- إعادة محاولة تلقائية بدون thread ID إذا فشلت
- لا مزيد من الرسائل الضائعة!

### 3. دعم Markdown Spoilers
```markdown
||نص مخفي||  ← يعمل الآن بشكل صحيح!
```

### 4. إصلاح قيود الأوامر
- تقليص تسجيل الأوامر إلى 100 فقط
- تجنب خطأ `BOT_COMMANDS_TOO_MUCH` عند البدء
- بداية أسرع وأكثر استقرارًا

### 5. تطابق DM أفضل
- مطابقة `allowFrom` مع user ID (ليس chat ID فقط)
- سجلات أوضح للـ pairing
- دقة أعلى في تحديد المستخدمين المصرّح لهم

## 🚀 تحسينات Gateway والاستقرار

### إصلاح "فقدان الذاكرة بعد الضغط"
**المشكلة**: كانت الـ Agents تنسى المحادثات بعد Compaction.

**الحل**: حفظ سلسلة `parentId` بشكل صحيح، الآن Agents تتذكر كل شيء! 🧠

```javascript
// قبل: بعد compaction ← ذاكرة فارغة ❌
// بعد: بعد compaction ← ذاكرة كاملة ✅
```

### استرداد تلقائي من Context Overflow
عندما تكون نتائج الأدوات كبيرة جدًا:
- **Capping استباقي**: قص النتائج قبل إرسالها
- **Truncation احتياطي**: اختصار تلقائي إذا تجاوزت الحد
- **لا مزيد من التوقف**: استمرار المحادثة بسلاسة

### Cron Jobs أكثر موثوقية
- مشاركة مسار Announce الموحد
- حفظ metrics عبر إعادات المحاولة
- جدولة أكثر دقة

## 🧩 تحسينات Tools والأدوات

### Web Search Cache الذكي
- **مفاتيح cache فريدة** لكل مزود (Perplexity, Grok, Brave)
- **تطبيع Model IDs**: دعم أفضل لـ Perplexity Direct
- نتائج أسرع وأكثر دقة

### Model Failover التلقائي
```javascript
// قبل: HTTP 400 ← توقف ❌
// بعد: HTTP 400 ← محاولة model بديل ✅
```

الآن أوبن كلاو (OpenClaw) يحاول Models بديلة تلقائيًا عند فشل النموذج الأساسي.

### Exec Approvals آمنة أكثر
الأوامر المعاد توجيهها تُعرض بخط monospace:

```
قبل: sudo rm -rf / ← صعب القراءة
بعد: `sudo rm -rf /` ← واضح وآمن
```

## 🧠 تحسينات Memory والأداء

### Voyage Embeddings محسّنة
- ضبط `input_type` للحصول على نتائج استرجاع أفضل
- دقة أعلى في البحث الدلالي

### إعادة استخدام Model Cache
بدلاً من تحميل نموذج لكل Agent:
- **مشاركة Cache** بين جميع الـ Agents
- **توفير الذاكرة**: تقليل استهلاك RAM
- **بداية أسرع**: لا مزيد من التحميل المتكرر

## الأسئلة الشائعة

**كيف أحدّث أوبن كلاو (OpenClaw) للإصدار 2026.2.9؟**
على macOS استخدم `brew upgrade openclaw`، أو `npm update -g openclaw` للتثبيت عبر npm. على Docker استخدم `docker pull openclaw/openclaw:2026.2.9`.

**هل تطبيق iOS متاح للجميع؟**
حالياً في مرحلة Alpha للمساهمين. سيتوفر قريباً للجمهور عبر TestFlight.

**هل التحديث يحافظ على إعداداتي وذاكرتي؟**
نعم، التحديث يحافظ على جميع الملفات في `~/clawd/` بما في ذلك SOUL.md وMEMORY.md والإعدادات.

**ما أهم تحسين في هذا الإصدار؟**
إصلاح فقدان الذاكرة بعد Compaction هو الأهم، يضمن أن مساعدك يتذكر المحادثات السابقة بشكل كامل.
